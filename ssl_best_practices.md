# SSL/TLS Best Practices

## ۱. تفاوت SSL و TLS

| ویژگی | SSL | TLS |
|-------|-----|-----|
| مخفف | Secure Sockets Layer | Transport Layer Security |
| وضعیت | منسوخ شده (Deprecated) | فعال و استاندارد فعلی |
| نسخه‌های معروف | SSL 2.0, SSL 3.0 | TLS 1.0, 1.1, 1.2, 1.3 |
| امنیت | آسیب‌پذیر (POODLE, DROWN) | امن، به‌ویژه TLS 1.3 |
| سرعت | کندتر | سریع‌تر (به‌ویژه TLS 1.3) |
| الگوریتم‌های رمزنگاری | قدیمی و ضعیف | مدرن و قوی |
| استفاده امروزی | نباید استفاده شود | استاندارد صنعت |

**نکته مهم:** اگرچه امروزه همه از اصطلاح "SSL" به‌صورت عامیانه استفاده می‌کنیم (مثل "SSL Certificate")، در واقعیت پروتکلی که استفاده می‌شود TLS است. SSL از سال ۲۰۱۵ توسط IETF رسماً منسوخ اعلام شده است.

## ۲. تفاوت Self-Signed و CA-Signed Certificates

### Self-Signed Certificate
- گواهینامه‌ای است که توسط **خود سرور** امضا شده، نه توسط یک مرجع معتبر
- مرورگرها آن را **نمی‌شناسند** و پیام هشدار نشان می‌دهند
- **رایگان** است و به‌سرعت قابل تولید است
- مناسب: محیط‌های توسعه (Development)، شبکه‌های داخلی، آزمایش و تست

### CA-Signed Certificate
- توسط یک **Certificate Authority (CA)** معتبر مانند Let's Encrypt، DigiCert، Comodo امضا شده
- مرورگرها آن را **می‌شناسند** و هشداری نشان نمی‌دهند
- می‌تواند رایگان (Let's Encrypt) یا پولی باشد
- مناسب: وب‌سایت‌های عمومی، سرویس‌های تجاری، هر چیزی که کاربر واقعی دارد

| معیار | Self-Signed | CA-Signed |
|-------|-------------|-----------|
| هزینه | رایگان | رایگان تا گران |
| اعتماد مرورگر | ✘ | ✔ |
| زمان صدور | فوری | دقیقه تا روز |
| مناسب برای پروداکشن | ✘ | ✔ |
| نیاز به تمدید دستی | بله | بله (یا خودکار) |

## ۳. پنج SSL/TLS Best Practice

### ۱. فقط از TLS 1.2 و TLS 1.3 استفاده کنید
```nginx
# Nginx config
ssl_protocols TLSv1.2 TLSv1.3;
```
نسخه‌های قدیمی‌تر (SSL 3.0، TLS 1.0، TLS 1.1) آسیب‌پذیری‌های شناخته‌شده دارند و باید غیرفعال شوند.

### ۲. از Cipher Suite های قوی استفاده کنید
```nginx
ssl_ciphers 'ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384';
ssl_prefer_server_ciphers on;
```
الگوریتم‌های قدیمی مانند RC4، DES، و 3DES ناامن هستند.

### ۳. HSTS را فعال کنید (HTTP Strict Transport Security)
```nginx
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload";
```
این header به مرورگر می‌گوید که همیشه از HTTPS استفاده کند و از حملات Downgrade جلوگیری می‌کند.

### ۴. گواهینامه را به‌موقع تمدید کنید
- گواهینامه‌ها معمولاً ۹۰ روز (Let's Encrypt) تا ۱ سال اعتبار دارند
- از **Certbot** برای تمدید خودکار استفاده کنید:
```bash
certbot renew --quiet
# یا cron job بگذارید
0 0 * * * certbot renew --quiet
```

### ۵. OCSP Stapling را فعال کنید
```nginx
ssl_stapling on;
ssl_stapling_verify on;
resolver 8.8.8.8 8.8.4.4 valid=300s;
```
این قابلیت وضعیت revocation گواهینامه را کش می‌کند و سرعت اتصال را بهبود می‌دهد.

## ۴. چرا HTTP ناامن است و HTTPS چگونه امنیت را تضمین می‌کند؟

### مشکلات HTTP

HTTP یک پروتکل **plaintext** است، یعنی تمام داده‌ها بدون رمزنگاری منتقل می‌شوند:

1. **Eavesdropping (شنود):** هر کسی در مسیر شبکه (مثل ISP، هکر در وای‌فای عمومی) می‌تواند محتوای ارتباط شما را ببیند
2. **Man-in-the-Middle (MITM):** مهاجم می‌تواند محتوا را **تغییر دهد** بدون اینکه شما بفهمید
3. **Tampering:** صفحات وب می‌توانند در حین انتقال دستکاری شوند (اضافه کردن تبلیغات، بدافزار و...)
4. **بدون تأیید هویت:** هیچ تضمینی وجود ندارد که با سرور واقعی ارتباط دارید

### چگونه HTTPS امنیت را تضمین می‌کند؟

HTTPS از TLS برای سه ضمانت اساسی استفاده می‌کند:

```
[کاربر] ←──── رمزنگاری شده ────→ [سرور]
           کسی نمی‌تواند بخواند
           کسی نمی‌تواند تغییر دهد
           هویت سرور تأیید شده
```

| ضمانت | توضیح |
|-------|-------|
| **Confidentiality (محرمانگی)** | داده‌ها رمزنگاری می‌شوند، فقط فرستنده و گیرنده می‌توانند بخوانند |
| **Integrity (یکپارچگی)** | با MAC (Message Authentication Code) تغییر داده‌ها تشخیص داده می‌شود |
| **Authentication (احراز هویت)** | گواهینامه SSL/TLS هویت سرور را تأیید می‌کند |

**فرایند TLS Handshake (ساده):**
1. کلاینت لیست الگوریتم‌های پشتیبانی‌شده را می‌فرستد
2. سرور گواهینامه و کلید عمومی خود را ارسال می‌کند
3. کلاینت گواهینامه را با CA verify می‌کند
4. یک **session key** مشترک به‌صورت امن توافق می‌شود
5. از این به بعد تمام ارتباط با این کلید رمزنگاری می‌شود

# CDN چیست و چگونه کار می‌کند؟

## ۱. CDN چیست؟

**CDN** مخفف **Content Delivery Network** است — شبکه‌ای از سرورهای توزیع‌شده در مکان‌های جغرافیایی مختلف که محتوای وب را به کاربران نزدیک‌تر تحویل می‌دهد.

به زبان ساده:
> به جای اینکه همه کاربران دنیا فایل‌هایشان را از یک سرور در آمریکا دریافت کنند، CDN یک کپی از محتوا را در نزدیک‌ترین سرور به هر کاربر ذخیره می‌کند.

### CDN چه کاری انجام می‌دهد؟
- **Cache** کردن محتوای استاتیک: تصاویر، CSS، JavaScript، ویدیو، فونت‌ها
- **توزیع بار** (Load Distribution) بین سرورهای مختلف
- **محافظت** در برابر DDoS و حملات شبکه‌ای
- **بهینه‌سازی** تحویل محتوا با فشرده‌سازی و caching هوشمند

**نمونه‌های CDN محبوب:** Cloudflare، Akamai، AWS CloudFront، Fastly، jsDelivr

## ۲. CDN چگونه Latency را کاهش می‌دهد؟

### مشکل بدون CDN
```
کاربر در تهران → درخواست → سرور در نیویورک (~10,000 km)
                                ↓
                        زمان رفت و برگشت: ~150-200ms
```

### با CDN
```
کاربر در تهران → درخواست → Edge Server در فرانکفورت (~3,000 km)
                                ↓
                        زمان رفت و برگشت: ~30-50ms
```

### دلایل کاهش Latency توسط CDN

**۱. مجاورت جغرافیایی (Geographic Proximity)**
- محتوا از نزدیک‌ترین نقطه (Edge Server) تحویل داده می‌شود
- مسافت فیزیکی کمتر = زمان انتقال کمتر

**۲. Caching هوشمند**
- فایل‌های پرمصرف در حافظه (RAM) Edge Server نگه داشته می‌شوند
- نیازی به رفتن به سرور اصلی نیست

**۳. بهینه‌سازی مسیر شبکه**
- CDNها از شبکه‌های خصوصی و بهینه‌شده بین سرورهای خودشان استفاده می‌کنند
- از شلوغی اینترنت عمومی اجتناب می‌کنند

**۴. HTTP/2 و HTTP/3**
- CDNهای مدرن از پروتکل‌های جدیدتر و سریع‌تر پشتیبانی می‌کنند

**۵. فشرده‌سازی (Compression)**
- فایل‌ها با Gzip یا Brotli فشرده می‌شوند → انتقال سریع‌تر

## ۳. تفاوت Origin Server و Edge Server

### Origin Server (سرور اصلی)
- سرور **اصلی و اصیل** که محتوای واقعی روی آن قرار دارد
- معمولاً یک یا چند سرور در یک مکان جغرافیایی مشخص
- مسئول پردازش منطق برنامه، دیتابیس و تولید محتوای دینامیک
- مستقیماً در معرض ترافیک کاربران قرار نمی‌گیرد (CDN جلوی آن است)

### Edge Server (سرور لبه)
- سرورهایی که در **نقاط مختلف جهان** توزیع شده‌اند (PoP - Point of Presence)
- محتوای **کش‌شده** از Origin Server را نگه می‌دارند
- اولین نقطه‌ای هستند که درخواست کاربر به آن می‌رسد
- در صورت cache miss، درخواست را به Origin می‌فرستند

### جریان کار

```
┌──────────┐    Cache HIT     ┌─────────────┐
│  کاربر   │ ←──────────────  │ Edge Server │
│  تهران   │                  │ فرانکفورت  │
└──────────┘                  └──────┬──────┘
                                     │ Cache MISS
                                     ↓
                              ┌─────────────────┐
                              │  Origin Server  │
                              │   نیویورک       │
                              └─────────────────┘
```

| ویژگی | Origin Server | Edge Server |
|-------|--------------|-------------|
| تعداد | کم (۱ تا چند) | زیاد (صدها در جهان) |
| محتوا | همیشه به‌روز | کش‌شده (ممکن است قدیمی باشد) |
| پردازش | منطق کامل برنامه | فقط serve محتوا |
| نزدیکی به کاربر | دور | نزدیک |
| فشار ترافیکی | کم (CDN جلوگیری می‌کند) | بالا |
| نمونه | سرور اصلی شرکت | Cloudflare PoP |
